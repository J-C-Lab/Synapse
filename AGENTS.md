# Repository Guidelines

## Project Structure

This is a pnpm workspace with the Electron desktop app at the repository root and the Fumadocs site in `docs/`.

- `src/main/` Electron main process, IPC registration, custom `app://` protocol, CSP, window lifecycle.
- `src/preload/` contextBridge surface and renderer-visible Electron types.
- `src/renderer/` Vite React renderer.
- `src/renderer/src/components/ui/` vendored shadcn/ui components. Do not add tests here.
- `resources/` electron-builder icons and build resources.
- `docs/` separate Next.js/Fumadocs documentation site.
- `DesKit/` project research and design documents.

## Coding Rules

- TypeScript strict mode.
- Renderer imports use `@/*` for `src/renderer/src/*`.
- Main-process imports may use `@main/*`; preload imports may use `@preload/*`.
- IPC should follow the existing pattern: pure handler in `src/main/ipc/`, explicit `ipcMain.handle`, typed preload bridge, renderer wrapper in `src/renderer/src/lib/electron.ts`.
- Electron security settings must keep `contextIsolation: true`, `sandbox: true`, `nodeIntegration: false`, and `webviewTag: false`.
- Keep `src/renderer/src/components/ui/` treated as third-party shadcn code.

## Testing

- Vitest is the unit test runner.
- Co-locate `*.test.ts` and `*.test.tsx` next to source files.
- Do not add test files inside `src/renderer/src/components/ui/`.

## Docs Site

- Docs imports should use `@/lib/source`.
- Use `fumadocs-ui/provider/next` for `RootProvider`.
- `docs/.source/` is generated by `pnpm docs:dev` or `pnpm docs:build`.
